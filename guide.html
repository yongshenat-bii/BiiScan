End-to-End Workflow

Phase 1: Prepare (Laptop)

1 Generate QR Bundle
  1. Open Generator on your laptop
  2. Upload CSV - drag or click to upload your participant list
  3. Configure fields:
    • Primary field - main identifier shown during scan (e.g. Name)
    • Display fields - extra fields shown in confirm popup (e.g. Email, Company)
  4. Set a passcode - this protects the session (share only with your team)
  5. Review & Export:
    • Download bundle.json (required for scanner)
    • Optional: QR Sheet PDF, individual receipt PNGs, email-links CSV
↓

Phase 2: Create Session (Admin Phone)

2 Admin creates the scanning session
  1. Open Scanner on your phone
  2. Enter your name (for audit trail)
  3. Tap "Create new session" at the bottom
  4. Select max devices allowed (default 6)
  5. Load the bundle.json file you exported
  6. A 6-character session code will be generated - share this with your team
The person who creates the session is automatically the Admin and can kick devices, view full logs, and export audit data.
↓

Phase 3: Team Joins (Scanner Phones)

3 Scanners join the session
  1. Open Scanner on each phone
  2. Enter your name
  3. Enter the session code shared by Admin
  4. Tap Join Session
  5. Enter the passcode when prompted
  6. Allow camera access when asked
Each scanner only sees their own scan count (not the full participant list) for privacy.
↓

Phase 4: Scan (Event Day)

4 Scan QR codes
  1. Point camera at a participant's QR code
  2. A confirmation popup appears showing the participant's details
  3. Tap Confirm to check them in
  4. If already scanned, it shows: Already collected - Name with date, time, and who scanned it
All phones sync in real-time. No two phones can double-scan the same QR.
↓

Phase 5: Monitor (Operator Laptop/Tablet)

5 Dashboard for on-site operations
  1. Open Dashboard
  2. Enter session code + passcode
  3. Live view of:
    • Total / Checked in / Pending / Progress %
    • Scans per person (device chips)
    • Connected devices count
  4. Search by name, email, or mobile
  5. Click any row to see full details + QR code
  6. Manual check-in / uncheck for walk-ins or corrections
  7. Export CSV at any time

Super Admin Panel

  1. Open Admin Panel
  2. Enter master password: 8118
  3. View all sessions with live status (active / idle / dead)
  4. Click any session to see details, online devices, scan stats
  5. Actions: Backup CSV, End Session (kick all), Delete Session
  6. Bulk: Backup All, Delete Old Sessions (>7 days with no devices)

Sending QR Codes via Email

  1. In the Generator, after exporting the bundle, scroll to Email Package
  2. Set the base URL to: https://yongshenat-bii.github.io/BiiScan/qr-receipt.html
  3. Select the email column from your CSV
  4. Click Export email-links.csv
  5. The exported CSV contains each participant's email + unique QR receipt link
  6. Use your email sender tool to mail each link to the participant
Each link is self-contained - no server or database needed. The participant sees a receipt page with their QR code and details.

Tips & Troubleshooting

  • Camera not working? - Must use HTTPS. Make sure you're accessing via the GitHub Pages URL, not a local file.
  • Device count shows 0? - Happens when phone sleeps. Unlock and it auto-recovers within seconds.
  • Admin lost access? - Rejoin the session with the same device. Admin status is tied to device ID and auto-restores.
  • Session timeout? - Sessions persist in Firebase indefinitely. No timeout during events. Data stays for 30 days (Firebase test mode).
  • Max devices? - Up to 6 phones per session. Configurable when creating the session.
  • CSV has special characters? - Handled automatically. Column names with /, ., # etc. are sanitized for Firebase.
  • QR too complex to scan? - QR codes only contain {"id":1}, kept minimal for fast scanning.
